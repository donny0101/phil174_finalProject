Engineerig teams will have to learn from this clearly terrible situation that checks and balances must be in place whereever machine learning is used. One problem with the Dutch System was how the there was no officiating team to monitor how the algorithm was performing on user data and thus the serious reprocussions weren't seen until years later. When a machine learning algorithm is used, a team must be in place to monitor and update the model for however long it is used in the system to adjust for changes when there are obvious problems with how the model is performing. This team must be knowledageable in how the model works, including coding and programming skills so that they can look through the code to find out why certain outputs are occuring based on the input data. For example, in the the case of the child benefits scandal the algorithm was miscorrectly giving individuals with low incomes or of a minority race or nation automatically received significant marks which would flag many as risky for parameters out of their control. The most ironic part is that the people most disproportionately affected are those that are the most in need of receiving these childcare benefits.
